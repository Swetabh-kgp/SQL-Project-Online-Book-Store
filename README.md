# Online Book Store SQL Analysis Project
## Project Overview
This project is based on an Online Book Store database created using PostgreSQL.  
The project analyzes books, customers, and order data to generate useful business insights.
## Tools Used
- PostgreSQL
- SQL
- CSV Files
## Dataset Tables
The project contains three main tables:
1. **Books**
   - Stores book details such as title, author, genre, price, published year, and stock.
2. **Customers**
   - Stores customer details such as name, email, city, and country.
3. **Orders**
   - Stores order details such as customer ID, book ID, order date, quantity, and total amount.
## SQL Analysis Performed
### Basic Queries
- Retrieved books by genre
- Found books published after a specific year
- Listed customers from a specific country
- Filtered orders by date
- Calculated total stock
- Found most expensive and lowest stock books
- Calculated total revenue
### Advanced Queries
- Total books sold by genre
- Average price by genre
- Customers with multiple orders
- Most frequently ordered book
- Top expensive books by genre
- Quantity sold by each author
- Highest spending customers
- Remaining stock after orders
## Key Skills Used
- Database creation
- Data importing from CSV
- SQL filtering
- Joins
- Group By
- Having clause
- Aggregate functions
- Order By and Limit
- Business data analysis
## Project Outcome
This project helped me practice SQL concepts and analyze business data for an online bookstore.  
It shows how SQL can be used to track sales, customer behavior, inventory, and revenue.
